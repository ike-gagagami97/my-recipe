# Cursor skills

Project skills for **Cursor** agents. Each skill is a folder with `SKILL.md` (YAML frontmatter + body).

Only the name/description need to be discoverable up front; the body is for when the skill matches the task.

| Skill | Use when |
| --- | --- |
| [`recipe-feature`](./recipe-feature/SKILL.md) | Recipe CRUD / App Router feature work |
| [`supabase-migration`](./supabase-migration/SKILL.md) | Tables, RLS, grants, seeds |
| [`ui-design`](./ui-design/SKILL.md) | Landing and visual design |
| [`verify-frontend-change`](./verify-frontend-change/SKILL.md) | Before declaring any UI change done |

See [`docs/development/steering.md`](../../docs/development/steering.md) for when to use a skill vs a rule vs an agent.
