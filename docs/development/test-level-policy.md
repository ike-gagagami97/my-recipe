# Test level policy

PR ごとに「どこまで検証するか」を決める方針。

- **レベル定義**: 合意済み（下記）
- **必須の付け方**: **ルール表（案 R）** で試す。足りなければ後からスコア制を検討
- **改善**: 開発を進めながらサブタイプや必須レベルは見直してよい（変更時はこのファイルと changelog を更新）
- 関連: [`workflow.md`](./workflow.md) の④-3

---

## レベル定義（合意）

積み上げ「全部やれ」ではなく、**ベースライン + 追加ゲート**として組み合わせる。

| Level | 名前 | 環境 | 実行者 | 内容 |
| --- | --- | --- | --- | --- |
| **L0** | 静的検証 | CI / local | エージェント / CI | `lint`、`build`（型チェック含む）、（将来）ユニットテスト |
| **L1** | 動的自動検証 | local（将来は CI も） | エージェント / CI | リポジトリに存在する E2E 等の自動テストを実行（**回帰**） |
| **L2** | エージェント検証 | local | エージェント（Playwright MCP 等） | feature doc の §5 Gherkin / §6 をブラウザ操作で確認 |
| **L3** | Preview エージェント検証 | Vercel Preview | エージェント | L2 相当を Preview URL で再実行（環境・env 込み） |
| **L4** | 人間受け入れ | Preview | 人間（QA） | 手動で受け入れ判断・最終確認 |

### 補足

- **L0**: ユニットが無い変更では unit は N/A
- **L1 と L2 は併存**: 回帰は L1、新規・未整備は L2。E2E が無い間の L1 は「該当テストがあるときのみ」
- **L3**: env / デプロイ設定など **Preview でしか分からないリスクがあるときだけ**必須（下記）
- **L4**: 常時必須ではない。ただし **新規画面・新規フローでは必須**。「振る舞い不変」の一次判定は **PR 作成者**（QA は覆せる）

## ルール表（試行・案 R）

PR 作成者が変更タイプを選び、必須レベルを PR 本文に書く。複数タイプに当てはまるときは **より重い方**を採用する。

### 一般

| 変更タイプ | 必須 | 任意 / 条件付き |
| --- | --- | --- |
| **docs / skills / 文言のみ**（振る舞い不変） | **L0** | — |
| **既存画面の軽い挙動変更** | L0 + L2 | L1（該当 E2E があれば実行） |
| **新規画面・新規フロー** | L0 + L2 + **L4** | L1（あれば） |
| **DB 読取の追加**（migration なし） | L0 + L2 | L1（あれば） |
| **migration / DB 書込** | L0 + L2 | L1（あれば）。L3 は「L3 条件」に該当するとき必須 |
| **env / デプロイ設定の変更** | L0 + **L3** | 画面振る舞いも変わるなら +L2。新規フローならさらに +L4 |

### 認証・権限・削除（H3 サブタイプ）

一括にせずサブタイプで選ぶ。今のアプリに認証が無くても、将来用にこの表を正とする。

| サブタイプ | 必須 | 任意 / 条件付き |
| --- | --- | --- |
| **ログイン・ログアウト・セッション** | L0 + L2 + **L4** | L1（あれば）。L3 は条件時 |
| **RLS / 権限の拡大**（より多く見れる・書ける） | L0 + L2 + **L4** | L1（あれば）。L3 は条件時 |
| **権限の表示・コピーのみ** | 振る舞い不変なら **L0**、挙動に触るなら L0 + L2 | — |
| **論理削除・取り消し可能削除** | L0 + L2 | L4 任意。L1（あれば） |
| **取り消し不可の物理削除・一括削除** | L0 + L2 + **L4** | L1（あれば） |

当てはまらない新しいパターンが出たら、無理に既存行へ押し込めず **サブタイプを追加**してこの表を更新する。

### L3 を必須にする条件

次の **いずれか**に当てはまるとき L3 必須:

1. `.env` / Vercel の Environment Variables の追加・変更が必要
2. デプロイ設定・リダイレクト・ドメイン・ヘッダなど、local と Preview で差が出やすい変更
3. 「local では足りず Preview でしか確認できない」と PR 作成者が判断したとき

当てはまらなければ L3 は不要（任意でやってよい）。

### 振る舞い不変の扱い

- 一次判定: **PR 作成者**
- docs / 文言のみで振る舞い不変 → **L0 のみ**
- PR 本文に一言書く（例: `振る舞い不変: 文言修正のみ → L0`）
- QA は覆せる

### 該当 E2E が無いとき（L1）

必須セットから L1 を外してよい。無理にテストを新設しない（別判断）。

### Playwright MCP が使えないとき（L2）

PR に理由を書く。代替:

- L4 が必須のタイプ（新規フロー、ログイン、権限拡大、物理削除など）: **L4 必須のまま**
- それ以外: QA と相談して L4 に上げるか、手順書付きの手動メモで暫定可

### ルール表の改善

- 実 PR で「重すぎる / 軽すぎる / 行が無い」と感じたら、マージ後でもこのファイルを更新してよい
- 更新時は下記 changelog に一行残す
- スコア制への移行は、ルール表で何度も例外が出てから検討すればよい

## PR 本文への書き方（短く）

GitHub の PR 作成時は [`.github/PULL_REQUEST_TEMPLATE.md`](../../.github/PULL_REQUEST_TEMPLATE.md) が載る。  
Test level 欄は次の形で埋める。

```text
Test level: L0 + L2 + L4
Type: 新規画面・新規フロー（レシピ一覧）
振る舞い不変: no
L3: no（env/デプロイ変更なし）
```

認証例:

```text
Test level: L0 + L2 + L4
Type: ログイン・ログアウト・セッション
L3: no
```
