## 1. ひとことで言うと（必須）

メールアドレスとパスワードでログインし、アプリを使い終わったらログアウトできる。ログインしないとレシピの閲覧・操作は一切できない。

## 2. だれの・どんな場面か（必須）

| 項目 | 書くこと |
| --- | --- |
| だれ | 自分用にレシピを溜めたい個人（Supabase Dashboard で手動登録されたユーザー） |
| ユーザー課題 | 自分のレシピを他人に見られたくない。アプリを開いたときに認証なしで中身が見えてしまう |
| 提供価値 | ログインした本人だけがレシピを閲覧・操作できる。ログアウトすれば第三者がそのままブラウザを開いても中身に触れない |
| 使う場面 | アプリを開くとき（ログイン）、使い終わったとき（ログアウト） |

## 3. できること（必須）

- [ ] メールアドレスとパスワードを入力してログインできる
- [ ] ログインに失敗したとき（認証情報が誤り）、エラーメッセージが表示される
- [ ] ログイン済みの状態でログアウトできる
- [ ] ログインしていない状態でレシピ画面に直接アクセスするとログイン画面にリダイレクトされる

## 4. やらないこと（必須）

- サインアップ画面（ユーザーは Supabase Dashboard から手動作成）
- ソーシャルログイン（Google、GitHub 等）
- パスワードリセット・再発行フロー
- ユーザープロフィール編集
- セッション管理の詳細設定（タイムアウト時間の変更等）
- 多要素認証（MFA）

## 5. 操作の流れ（必須・Gherkin）

```gherkin
Feature: 認証（ログイン・ログアウト）
  ログインした本人だけがレシピを閲覧・操作できる

  Scenario: 正しい認証情報でログインできる
    Given ユーザーが Supabase Dashboard で作成済みである
    And ユーザーはログインしていない
    When ログイン画面でメールアドレスとパスワードを入力して送信する
    Then レシピ一覧画面（または保護されたページ）に遷移する
    And ログイン状態であることが画面から分かる（例: ログアウトボタンが見える）

  Scenario: 誤った認証情報でログインが失敗する
    Given ユーザーはログインしていない
    When ログイン画面で誤ったパスワードを入力して送信する
    Then ログイン画面に留まる
    And 認証に失敗したことが分かるエラーメッセージが表示される

  Scenario: ログアウトできる
    Given ユーザーはログイン済みである
    When ログアウトボタンを押す
    Then ログイン画面に遷移する
    And ログアウト後に一覧画面の URL を開くとログイン画面にリダイレクトされる

  Scenario: 未ログインでレシピ画面にアクセスするとリダイレクトされる
    Given ユーザーはログインしていない
    When レシピ一覧画面（/recipes）に直接アクセスする
    Then ログイン画面にリダイレクトされる
    And レシピの中身は表示されない
```

## 6. 受け入れ条件（必須）

### 合格条件

- [ ] §5 の各 Scenario を Preview またはローカルで確認した
- [ ] 正しい認証情報でログインするとレシピ一覧（または保護画面）に遷移する
- [ ] 誤った認証情報ではエラーメッセージが表示され、ログイン画面に留まる
- [ ] ログアウト後はログイン画面に遷移し、ブラウザの戻るボタンで保護ページに戻れない（またはリダイレクトされる）
- [ ] 未ログイン状態で `/recipes` 等の保護ページに直接アクセスするとログイン画面にリダイレクトされる
- [ ] レシピページはログイン後にしかアクセスできない（サーバーサイドで保護されていること。クライアントのみの保護は不可）

### 確認メモ（環境・範囲）

- 確認環境: ローカル / Preview
- 端末: デスクトップ優先（モバイル幅も基本確認）
- 回帰で触る画面: ランディング画面（ログインへの動線が変わる場合）
- その他メモ: テスト用ユーザーを Supabase Dashboard で事前作成してから確認する

## 7. 空・エラー・例外（任意）

| 場面 | 期待する見え方・挙動 |
| --- | --- |
| メールアドレスが未入力 | フォームバリデーションでエラー（送信前 or 送信後）。「メールアドレスを入力してください」等 |
| パスワードが未入力 | 同上 |
| 認証情報が誤り | 「メールアドレスまたはパスワードが正しくありません」等（存在確認の情報漏洩を避ける文言） |
| ネットワークエラー | 失敗したことが分かるメッセージ。真っ白・無限ローディングにはしない |
| すでにログイン済みでログイン画面にアクセス | レシピ一覧にリダイレクト（未決。ログイン画面をそのまま表示でも可） |

## 付録（任意・空欄のままでよい）

### 実装メモ（エージェント用）

- 想定ルート: `/login`（ログイン）、保護ページは proxy.ts（Next.js 16 の middleware 相当）でガード
- データ: Supabase Auth を使用。`recipes` テーブルの RLS で `auth.uid()` を使う前提
- 使う skills: `supabase-migration`（Auth 有効化）、`verify-frontend-change`、`nextjs`（proxy.ts でのリダイレクト）
- 注意: Next.js 16 では middleware は `proxy.ts`。`cookies()` / `headers()` は async
