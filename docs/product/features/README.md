# Product features

機能ごとの **振る舞いの契約**（feature doc）を置く場所です。

- テンプレート: [`_template.md`](./_template.md)
- 書き手の主想定: QA / プロダクト（技術詳細は必須ではない）
- 開発フロー上の位置: [`../development/workflow.md`](../development/workflow.md) の②

## 必須セクション（§1〜6）

1. ひとことで言うと  
2. だれの・どんな場面か（ユーザー課題 + 提供価値）  
3. できること  
4. やらないこと  
5. 操作の流れ（Gherkin）  
6. 受け入れ条件（確認メモもここ。別欄に分けない）

§7 以降と付録は任意（書いてあればレビュー対象）。

新しい機能を始めるとき:

1. `_template.md` をコピーして `<feature-name>.md` を作る
2. 人間が §1〜6 を承認する
3. Issue からリンクし、エージェント依頼ではそのパスだけ渡す
