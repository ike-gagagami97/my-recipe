## 1. ひとことで言うと（必須）

保存済みのレシピを、一覧画面でタイトルから眺められる。

## 2. だれの・どんな場面か（必須）

| 項目 | 書くこと |
| --- | --- |
| だれ | 自分用にレシピを溜めたい個人 |
| ユーザー課題 | メモやスクショに散らばって、「今夜何作ろう」のときに候補を一覧できない |
| 提供価値 | アプリを開けばレシピのタイトルが一覧でき、献立の候補をすぐ把握できる |
| 使う場面 | 献立を考えるとき、買い物前に「家にあるレシピ」をざっと見たいとき |

## 3. できること（必須）

- [ ] レシピ一覧画面を開く
- [ ] 保存済みレシピのタイトルを一覧で見る
- [ ] レシピが0件のとき、空であることが分かる

## 4. やらないこと（必須）

- レシピの追加・編集・削除（別 feature）
- レシピ詳細画面への遷移（別 feature。一覧のみ）
- 並び替え UI（新しい順固定）
- 画像・サムネイル表示

## 5. 操作の流れ（必須・Gherkin）

```gherkin
Feature: レシピ一覧
  保存済みのレシピをタイトル一覧で見られる

  Scenario: レシピがあるときタイトルが一覧に出る
    Given レシピが1件以上保存されている
    When レシピ一覧画面を開く
    Then 各レシピのタイトルが一覧に見える
    And タイトルは新しいものが上に並んでいる

  Scenario: レシピが0件のとき空の案内が出る
    Given レシピが1件もない
    When レシピ一覧画面を開く
    Then まだレシピがないことが分かる案内が見える
    And レシピのタイトル行は出ない
```

## 6. 受け入れ条件（必須）

### 合格条件

- [ ] §5 の各 Scenario を Preview またはローカルで確認した
- [ ] 一覧に出るのはタイトルである（本文全文を一覧に並べない）
- [ ] 0件時に「壊れている」ように見えず、空の案内だと分かる
- [ ] ログイン済みユーザーの自分のレシピのみ表示される（他ユーザーのレシピは見えない）

### 確認メモ（環境・範囲）

- 確認環境: ローカル / Preview
- 端末: モバイル幅も見る
- 回帰で触る画面: ランディング（一覧への行き方が分かる場合）
- その他メモ: データはシードまたは手動投入で「1件以上」「0件」の両方を用意して確認する

---

## 7. 空・エラー・例外（任意）

| 場面 | 期待する見え方・挙動 |
| --- | --- |
| データが0件 | 「まだレシピがありません」などの案内文を表示。追加ボタンは出さない（追加は別 feature） |
| 一覧の取得に失敗 | エラーメッセージを表示（真っ白・無限ロードにしない） |
| 未ログイン | ログイン画面へリダイレクト（既存の認証フローに委ねる） |

## 8. オープンクエスチョン（任意）

- [x] 0件時に「追加へ」の導線を出すか？ → **出さない**（追加は別 feature のため）
- [x] 並びは「新しい順」で確定でよいか？ → **確定**（§5 Gherkin どおり）
- [x] 一覧の入口 URL / ナビ文言は日本語「レシピ」でよいか？ → **URL は `/recipes`、見出しは「レシピ一覧」**

## 付録（任意・空欄のままでよい）

### 実装メモ（エージェント用）

- ルート: `/recipes`（既存の `src/app/recipes/page.tsx` を活用）
- DB テーブル: `public.recipes`（`id`, `user_id`, `title`, `created_at`）
  - `user_id` は `auth.users.id` への外部キー
  - RLS: `auth.uid() = user_id` で自分のレシピのみ参照可
  - GRANT: `authenticated` ロールのみ（anon は不可）
  - 並び: `created_at DESC`（新しい順）
- 使う skills: `recipe-feature`, `supabase-migration`, `verify-frontend-change`
